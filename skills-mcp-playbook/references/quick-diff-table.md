# Skill vs MCP クイック対比表

| 項目 | Skill | MCP |
| --- | --- | --- |
| 主目的 | ワークフロー手順・品質基準・完了条件を定義 | 外部システムへの接続・データ取得/操作を提供 |
| 典型トリガー | 「どう進めるか」「何を先に見るか」「どうまとめるか」 | 「どこから/何を取るか」「どこへ書き込むか」 |
| 入出力指示 | シーケンス、優先順位、出力の型・セクション | API/ツールの呼び方と返却形式（JSON等） |
| 責務境界 | プロセスと表現。整形/検証ロジックを担う | 接続とI/O契約。ツール固有パラメータを担う |
| 衝突回避の原則 | フォーマット指示はSkillに寄せ、MCPは生データ返却 | 返却形式を固定するならSkill側で後工程整形を前提にする |
| 組み合わせ例 | 複数MCPを順序付けし、例外時のフォールバックを定義 | どのMCPをいつ使うかをSkillから呼び出される |

迷ったときは「アクセス/操作＝MCP」「やり方/品質＝Skill」で切り分ける。
