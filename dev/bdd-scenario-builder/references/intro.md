# BDD Overview (Facts)

- Behavior-Driven Development (BDD): 利用者・業務の言葉で振る舞いを例示し、テスト可能な形で合意して開発を進める考え方。外側（利用者視点）から内側へ進め、自動化を前提とする。  
  - Wikipedia: https://en.wikipedia.org/wiki/Behavior-driven_development  
  - Dan North (behave): https://behave.readthedocs.io/en/latest/philosophy/  
- Given-When-Then (GWT): 前提・操作・結果でシナリオを記述する型。  
  - Martin Fowler: https://martinfowler.com/bliki/GivenWhenThen.html  
- Gherkin: Cucumber などで使われるシナリオ記述言語（Feature/Rule/Scenario/Tags）。  
  - Reference: https://cucumber.io/docs/gherkin/reference/  
- TDDとの関係: TDDを土台にしつつ、業務言語と例を中心に認識合わせを進める発展形として説明されることがある。  
  - Martin Fowler (TDD): https://martinfowler.com/bliki/TestDrivenDevelopment.html
