# ヒアリング質問
- ゴール/成功条件は何か（未読消化/探索/比較/発見/記録/監視など）。優先順位は。
- 行動が時系列順序に依存するか、鮮度がどれほど重要か。
- 情報をカテゴリ・プロジェクト・人・状態などで塊化できるか。塊は5〜12程度に収まるか。
- 重要/未読/期限の偏りがあるか。重要だけ拾いたいか。
- 複数コンテキストを並行で行き来するか。一つの塊で完結するか。
- 1回の滞在時間と頻度はどれくらいか（短時間連打 vs 腰を据える）。
- 主要デバイスは何か（モバイル優先/デスクトップ広め/マルチウィンドウ）。
- 既存の痛点は何か（スクロール苦、探し物、切替の多さ、未読の山など）。
- 位置や塊で覚えたい/束ねたいニーズはあるか。

# 回答→判定マッピング（目安）
- 時系列依存が強い + 鮮度最優先 + 短時間連打 → タイムライン中心
- 塊化できる + 並行コンテキスト + 位置記憶したい → トポロジー中心
- 重要/未読が偏る → タイムラインならセクション/ピン、トポロジーなら優先領域＋ローカルタイムライン
- 目的が塊内で完結する → トポロジーで閉じる（塊内に必要ならタイムライン）
- モバイル狭小 → タイムライン主体＋ジャンプ/フィルタ、トポロジーは簡略
- デスクトップ広め＋複数文脈 → トポロジー強め＋コンテキスト内タイムライン
- 滞在時間が長い/比較が多い → トポロジー寄り（並列閲覧しやすくする）
