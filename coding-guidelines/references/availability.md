## 可用性 / クラッシュ防止
- ★3: ガードで前提を明示し、想定外入力は早期returnまたは例外変換。null/Noneチェックを最初に書く。
- ★3: 外部I/Oやパース失敗を握りつぶさず、ユーザー影響が最小になる代替パスを取る（フェイルソフト）。
- ★2: 不正データがあっても可能な限り処理を継続し、ログに文脈を残す（入力値、ID、簡潔な原因）。
- ★2: 入力バリデーションは境界値を含めて一カ所にまとめる（APIならリクエスト層で集約）。
- ★1: レイテンシの長い処理はタイムアウトを設定し、キャンセル可能にする。

### 簡易チェック（gate向け）
- Fail: 前提不成立時にクラッシュ/無限ループになる経路がある。
- Warn: 想定外入力の扱いが仕様化されていない、ログが不足。
- OK: ガードとバリデーションで異常系の出口が定義されている。

### 例（before/after）
- before: `return items[0]` （itemsが空で落ちる）
- after: `if not items: return default_item  # ★3 ガード`  
           `return items[0]`
